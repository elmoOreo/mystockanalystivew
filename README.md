# Stock Analyst Dashboard

A Flutter application designed to visualize the output of an AI-powered stock analysis engine. This dashboard provides a clean and intuitive interface to browse through analysis reports and the detailed logs generated by the underlying AI agents.

## Features

-   **Analyst Runs:** View a historical list of analysis runs, filterable by their execution timestamp.
-   **Detailed Reports:** For each run, browse detailed sentiment analysis cards for multiple stock tickers. Each card includes:
    -   Ticker symbol
    -   Sentiment (e.g., Bullish, Bearish)
    -   A confidence score (0-100)
    -   Detailed reasoning behind the analysis.
-   **Agent Logs:** Dive deep into the execution of each analysis run by viewing the step-by-step logs from the AI agents.
-   **Log Filtering:** Easily filter the agent logs by date and the specific agent that produced the log.
-   **Expandable Details:** View the raw JSON prompt and output for each agent step to understand the AI's process.
-   **Modern UI:** A sleek, dark-themed interface built for clarity and ease of use.

## Project Structure

The project is organized into a modular structure to promote separation of concerns and scalability:

```
lib/
├── config/         # Application-level configuration (e.g., API endpoints)
├── models/         # Data models (AnalystRun, ReportItem, AgentLog)
├── screens/        # Top-level screen widgets that orchestrate views
├── services/       # Business logic, primarily for API communication
├── views/          # UI components for specific tabs/parts of a screen
└── main.dart       # Main application entry point
```

## Prerequisites

Before you begin, ensure you have the following installed:

-   [Flutter SDK](https://flutter.dev/docs/get-started/install) (version 3.x or higher)
-   A running instance of the [Stock Analyst Backend Server](<#link-to-your-backend-repo>).

## Setup & Installation

1.  **Clone the repository:**
    ```bash
    git clone <your-repository-url>
    cd mystockanalystview
    ```

2.  **Install dependencies:**
    ```bash
    flutter pub get
    ```

3.  **Configure the API Endpoint:**
    Open the `lib/config/app_config.dart` file and update the `apiBaseUrl` to point to your running backend server.

    ```dart
    class AppConfig {
      // For Android emulator, use '10.0.2.2'.
      // For iOS simulator and web, 'localhost' or '127.0.0.1' usually works.
      static const String apiBaseUrl = 'http://localhost:3000/api';
    }
    ```

## Running the Application

Once the setup is complete and the backend server is running, you can launch the Flutter app:

```bash
flutter run
```

Select your desired device (iOS Simulator, Android Emulator, or a connected physical device) from the list to run the application.
